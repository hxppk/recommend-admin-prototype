## 1. 产品概述

### 1.1 产品定位
茶百道小程序点单页推荐系统的运营管理后台，用于运营人员配置和管理推荐系统的核心参数，覆盖**选品池 → 排序策略 → 策略组合 → 投放计划**四层架构。

### 1.2 目标用户
| 角色 | 职责 | 可操作范围 |
|------|------|-----------|
| 运营人员 | 日常选品、策略配置 | 创建和管理自己创建的选品池、策略、组合、计划 |
| 管理员 | 全局管理、优先级配置 | 所有操作 + 投放计划优先级配置 |
| 系统内置 | 自动维护的默认配置 | 不可编辑、不可删除 |

### 1.3 核心业务流程
```
商品库 → 选品池（定义候选范围） → 排序策略（定义排序规则）
        → 策略组合（多策略多坑位编排） → 投放计划（绑定时间/门店/人群生效）
        → 用户端推荐位展示
```

## 3. 整体架构

### 3.1 四层架构关系图

```
┌─────────────────────────────────────────────────────────┐
│                    投放计划 (Plan)                        │
│  时间范围 + 门店范围 + 人群范围 + 优先级 + ABTest           │
│  绑定 ↓                                                  │
│              ┌─────────────────────────────────────┐     │
│              │         策略组合 (Combination)        │     │
│              │  坑位1 → 策略A  |  坑位2 → 策略B      │     │
│              │  同品类最多连续展示数 / Session去重     │     │
│              └────────────┬────────────────────────┘     │
│                           ↓                              │
│         ┌─────────────────────────────────────┐          │
│         │         排序策略 (Strategy)           │          │
│         │  选品池 + 排序模式 + 兜底策略          │          │
│         │  HOT / NEW / MANUAL / PERSONALIZED    │          │
│         └────────────┬────────────────────────┘          │
│                      ↓                                   │
│         ┌─────────────────────────────────────┐          │
│         │         商品选品池 (Pool)              │          │
│         │  商品集合 + 启停状态                   │          │
│         │  SYSTEM 系统池 / CUSTOM 自定义池       │          │
│         └────────────┬────────────────────────┘          │
│                      ↓                                   │
│              商品库 (Product)                              │
│              SPU + 品类 + 价格 + 状态                      │
└─────────────────────────────────────────────────────────┘
```

### 3.2 推荐引擎决策流程
```
请求 → 门店过滤 → 人群匹配 → 时间校验 → 优先级排序
     → 命中最高优先级计划 → 获取策略组合
     → 按坑位依次执行策略 → 去重 → 品类连续限制 → 输出商品列表
     → 若未命中任何计划 → 使用兜底热销策略
```

## 5. 第一层：商品选品池

### 5.1 业务说明
选品池定义"哪些商品可以被推荐"。系统内置一个"ALL 全量池"（SYSTEM 类型），包含全部商品，不可删除。运营人员可创建自定义选品池（CUSTOM 类型），手动添加或批量导入商品。

### 5.2 数据结构（Pool）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识，格式 `pool-xxxxxx` |
| name | string | 选品池名称，1-30 字符 |
| description | string | 描述，最多 200 字符 |
| status | `ACTIVE` / `INACTIVE` | 启停状态 |
| kind | `SYSTEM` / `CUSTOM` | 系统内置 / 自定义 |
| productIds | string[] | 关联的商品 ID 列表 |
| productAddedTimes | Record<string, string> | 每个商品的添加时间 |
| createdAt | string | 创建时间 |
| createdBy | string | 创建人 |
| updatedAt | string \| null | 最后更新时间 |
| updatedBy | string \| null | 最后更新人 |

### 5.3 页面：选品池列表 `/pools`

**布局**：卡片网格（响应式：xl 3列 / md 2列 / sm 1列），系统池置顶

**筛选与搜索**：
- 搜索框：按选品池名称模糊搜索，300ms 防抖
- 新建选品池按钮

**卡片内容**：
- 名称 + SYSTEM 标签
- 状态 Tag（启用/停用）
- 商品数
- 被使用（引用该池的策略数量）
- 描述（省略 2 行）
- 创建信息和更新时间

**新建选品池交互**：
- 侧滑 Drawer，Form 包含：
  - 选品池名称（必填，1-30 字符，纯空格校验，名称唯一性校验）
  - 描述（选填，最多 200 字符）
  - 初始状态 Switch（默认启用）
- 按钮：「创建」→ 返回列表；「创建并添加商品」→ 跳转详情页

**空状态**：`<Empty description="还没有选品池">` + 创建按钮

### 5.4 页面：选品池详情 `/pools/:id`

**面包屑**：推荐系统 / 商品选品池 / 详情

**区域一：基础信息**
- Descriptions 展示：名称、状态、商品数量、创建时间、描述、创建人、最新维护
- 可编辑模式（创建人可编辑）：内联 Input/TextArea 修改
- 操作按钮：编辑基础信息、启用/停用、删除
  - 删除需确认名称输入（防误删）
  - 被策略引用的池不可删除

**区域二：商品清单**
- Table 展示：商品头像+名称、SPU ID、品类、状态、价格、添加时间
- 搜索：按商品名/SPU ID 模糊搜索
- 批量选择 + 批量移除
- 单个移除（Popconfirm 确认）
- SYSTEM 池不可手动增删

**添加商品**：
- Drawer 侧滑，搜索可用商品列表，逐个添加

**批量导入**：
- Modal 弹窗，两个 Tab：
  - 文本粘贴：输入 SPU ID（逗号/空格/换行分隔，上限 500 个）
  - CSV 导入：拖拽上传 `.csv` 文件，解析第一列 SPU ID
- 实时解析结果：有效 / 重复 / 无效 三栏展示

**关联策略**（折叠面板）：
- 列出引用此池的排序策略，可跳转查看

## 6. 第二层：排序策略

### 6.1 业务说明
排序策略定义"商品如何排序"。每个策略关联一个选品池，选择一种排序模式，配置相应的参数，并可设置兜底策略。

### 6.2 数据结构（Strategy）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识，格式 `strategy-xxxxxx` |
| name | string | 策略名称，1-30 字符 |
| description | string | 描述，最多 200 字符 |
| poolId | string | 关联的选品池 ID |
| mode | `HOT` / `NEW` / `MANUAL` / `PERSONALIZED` | 排序模式 |
| status | `ACTIVE` / `INACTIVE` | 启停状态 |
| sortDimension | `SALES_COUNT` / `SALES_AMOUNT` | 热销排序维度 |
| timeWindow | `7D` / `14D` / `30D` | 时间窗口 |
| fallbackStrategyId | string \| null | 兜底策略 ID |
| manualProductIds | string[] | 人工定序商品列表 |
| filterUnavailable | boolean | 是否过滤下架商品（固定 true） |
| kind | `SYSTEM` / `CUSTOM` | 系统内置 / 自定义 |
| manualBoostItems | ManualBoostItem[] | 人工加权商品列表 |
| manualBoostEnabled | boolean | 是否开启人工加权 |
| createdAt / createdBy | string | 创建信息 |

**人工加权项（ManualBoostItem）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| productId | string | 商品 ID |
| weight | number | 加权系数，范围 [-1, 1]，步长 0.05 |
| startAt | string | 生效开始日期 |
| endAt | string | 生效结束日期 |

### 6.3 排序模式详解

| 模式 | 名称 | 排序规则 | 需配置参数 |
|------|------|---------|-----------|
| HOT | 热销排行 | 按销量/交易额自动排序 | 排序维度、时间窗口 |
| NEW | 新品曝光 | 按上新时间排序 | 时间窗口 |
| MANUAL | 人工定序 | 人工拖拽确定顺序 | 手动添加商品列表 |
| PERSONALIZED | 个性化推荐 | 系统 CTR/CVR 预估模型 | 无需配置（系统自动） |

### 6.4 页面：排序策略列表 `/strategies`

**筛选与搜索**：
- 搜索框：按策略名称模糊搜索
- 排序方式 Select：热销排行 / 新品曝光 / 人工定序
- 状态 Select：启用 / 停用
- 新建策略按钮

**表格列**：

| 列 | 内容 |
|----|------|
| 策略名称 | 点击跳转编辑页，SYSTEM 策略标注"系统"标签 |
| 排序方式 | Tag 显示模式（颜色区分） |
| 状态 | Tag 启用/停用 |
| 引用选品池 | 点击跳转选品池详情 |
| 被引用 | Tag 蓝色显示引用数量（如"2个"）+ 点击跳转策略组合 |
| 兜底策略 | 显示兜底策略名称 |
| 创建时间 | 格式化时间 |
| 操作 | 查看 / 启用/停用 / 复制 / 删除 |

**交互规则**：
- SYSTEM 策略不可删除、不可启停
- 被引用的策略不可删除
- 仅创建人可编辑/删除
- 系统策略置顶显示

**空状态**：`<Empty description="暂无数据" image={Empty.PRESENTED_IMAGE_SIMPLE} />`

### 6.5 页面：排序策略编辑 `/strategies/:id/edit`

**面包屑**：推荐系统 / 排序策略 / 编辑

**区域一：基础信息**
- 策略名称（必填，1-30 字符，校验规则：required + 长度限制）
- 选择选品池（必填，下拉显示池名 + 商品数）
- 描述（选填，最多 200 字符）

**区域二：排序模式**
- 4 个卡片式 Radio 选择，选中状态有蓝色边框 + 右上角对勾标记
- 鼠标悬停时边框高亮 + 阴影

**模式参数配置**：

- **HOT 热销排行**：
  - 排序维度（必填）：销量（杯数）/ 销售额（金额）
  - 时间窗口（必填）：近 7 天 / 近 14 天 / 近 30 天

- **NEW 新品曝光**：
  - 时间窗口（必填）：近 7 天 / 近 14 天 / 近 30 天

- **MANUAL 人工定序**：
  - 左侧：选品池商品列表（搜索 + 添加按钮）
  - 右侧：已选排序列表（支持拖拽排序 + 上移/下移 + 移除）
  - 商品展示：名称 + SPU + 品类 Tag + 价格
  - 空状态提示

- **PERSONALIZED 个性化推荐**：
  - 展示说明信息，无需额外配置

**区域三：人工加权**
- Switch 开关控制是否开启
- 开启后展示 Table：
  - 商品选择（下拉搜索）
  - 加权系数 InputNumber（-1 ~ 1）
  - 生效时间 DatePicker 范围
  - 删除按钮

**区域四：高级配置**
- 兜底策略选择（可选），下拉列出除当前策略外的所有策略
- 实时检测循环引用：若兜底策略形成环，显示红色错误提示，保存按钮禁用
- 过滤规则：系统自动过滤下架/不可售商品（固定开启）

**底部操作栏**：
- 取消 → 返回策略列表（新建策略时自动删除草稿）
- 保存 → 校验表单后保存并返回列表

**Mock 策略数据**：

| 策略 | 模式 | 选品池 | 兜底策略 | 场景 |
|------|------|--------|---------|------|
| 全量热销榜 | HOT | ALL 全量池 | 无 | 系统内置，所有策略的最终兜底 |
| 新品曝光优先 | NEW | 春季新品池 | 全量热销榜 | 新品推荐场景 |
| 咖啡人工定序 | MANUAL | 咖啡加购池 | 全量热销榜 | 运营精选咖啡 |
| 果茶热销补位 | HOT | 爆款果茶池 | 全量热销榜 | 果茶推荐补位 |

## 7. 第三层：策略组合

### 7.1 业务说明
策略组合将多个排序策略分配到不同的"坑位"中，形成多路召回+组合展示的推荐结果。例如首页第一个坑位用"全量热销"，第二个坑位用"果茶热销"。

### 7.2 数据结构（Combination）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识，格式 `comb-xxxxxx` |
| name | string | 组合名称 |
| status | `ACTIVE` / `INACTIVE` | 启停状态 |
| slots | CombinationSlot[] | 坑位列表，每个坑位绑定一个策略 |
| categoryLimit | number \| null | 同品类最多连续展示数（null = 不限制） |
| sessionDedup | boolean | Session 内去重（固定 true，不可修改） |
| createdAt / createdBy | string | 创建信息 |

**坑位（CombinationSlot）**：

| 字段 | 类型 | 说明 |
|------|------|------|

### 9.3 推荐引擎逻辑

**计划匹配流程**：
1. 过滤已结束的投放计划
2. 检查投放时间是否在生效范围内（基于 `NOW = '2026-04-15T12:00:00+08:00'`）
3. 检查资源位是否匹配（slotIds 包含当前选择的资源位）
4. 检查门店是否在范围内（ALL / REGION / STORE）
5. 检查人群是否匹配（ALL / SEGMENT）
6. 按优先级降序排列，取最高优先级的计划

**策略执行流程**：
1. 获取计划绑定的策略组合
2. 按坑位顺序依次执行策略
3. 每个策略：尝试策略本身 → 兜底链（fallbackStrategyId 链）→ 返回第一个未去重的商品
4. 全局去重：同一商品不重复展示（seen Set）
5. 品类连续限制：同品类不超过 categoryLimit（如 categoryLimit=2 则同品类最多连续 2 个）

**兜底流程**：
- 未命中任何计划时，使用"全量热销榜"策略兜底，取前 4 个热销商品
- 策略链形成环形引用时自动检测并提示

**用户画像推断**（Mock）：
- 基于 userId 的 Hash 值推断 Segment
- 含 "coffee" 的 userId 自动匹配"咖啡偏好用户"

## 11. ABTest 集成说明

### 11.1 火山引擎集成方式

系统不直接执行 ABTest 分流计算，仅在管理后台配置实验信息，运行时将配置透传给火山引擎。

**数据流**：
```
运营后台配置 Plan.abTest → 存储 experimentKey + groups
       → 推荐引擎运行时收到请求（userId + planId）
       → 根据 userId 哈希 + traffic 占比分配实验分组
       → 透传对应分组的 VID 到火山引擎
       → 火山引擎返回实验结果
```

### 11.2 分流规则
- 分流粒度：用户级（user_id）
- 同一门店内不同用户可能看到不同内容
- 分流算法：基于 `${plan.id}:${userId}` 的哈希值，对 100 取模，按各分组 traffic 累计占比分配
- 例如：基线组 50% → 哈希值 0-49 命中基线组，50-99 命中增强组

### 11.3 配置项
| 配置项 | 说明 |
|--------|------|
| experimentKey | 火山引擎实验标识，对应火山平台上的实验 ID |
| 分组名称 | 便于运营识别，不传给火山引擎 |
| VID | 火山引擎分组标识，对应实验配置中的 variation_id |
| traffic | 流量占比百分比，总和应为 100 |

### 11.4 v1.0 限制
- 不实际调用火山引擎 API，分流逻辑在推荐引擎本地模拟
- 不展示实验统计报表
- 不自动检测流量是否均衡
- 仅在管理后台完成配置存储

  - experimentKey: "order-reco-lunch"
  - 基线组: VID1001, traffic 50%
  - 增强组: VID1002, traffic 50%

### 12.4 状态管理
- `AdminStoreContext` 提供全局状态和操作方法
- 各实体的增删改查方法封装在 store 中：
  - createXxx() → 生成 ID + 创建默认值 + 返回新 ID
  - updateXxx(id, next) → 替换对应实体
  - copyXxx(id) → 深拷贝 + 重命名 + 返回新 ID
  - deleteXxx(id) → 从列表中移除（SYSTEM 类型不可删除）
- 支持 copy 操作（策略/组合/计划），复制时自动重命名为"xxx - 副本"
- 状态变更自动写入 localStorage

### 12.6 核心业务逻辑函数（domain.ts）

| 函数 | 用途 |
|------|------|
| `simulatePreview()` | 模拟推荐请求，返回完整推荐结果 + 诊断信息 |
| `detectStrategyCycle()` | 检测兜底策略是否形成循环引用 |
| `findPlanConflicts()` | 检测投放计划冲突（同优先级+同门店+同时间） |
| `estimateCombinationCoverage()` | 估算策略组合覆盖商品数 |
| `getStrategyProducts()` | 获取策略排序后的商品列表 |
| `getPoolProducts()` | 获取选品池关联的商品 |
| `getStrategyReferences()` | 查询策略被哪些组合引用 |
| `getPoolReferences()` | 查询池被哪些策略引用 |
| `getPlanReferences()` | 查询组合被哪些计划引用 |
| `inferSegments()` | 根据 userId 推断用户人群画像 |
| `getTimeStatus()` | 计算计划当前状态（投放中/待生效/已结束等） |
| `assignAbGroup()` | 根据 userId 分配 ABTest 实验分组 |
