# PRD: 推荐系统运营管理后台（四层架构）

## 1. 项目背景

### 1.1 需求简介
在小程序点单页推荐系统中，构建一套完整的运营管理后台，覆盖四层架构（选品池→排序策略→策略组合→投放计划）的全部 CRUD 操作，以及效果监控和全链路预览工具。Q1 目标：跑通配置→上线→看数据的最小闭环。

### 1.2 四层架构总览
- **第一层：选品池（召回池）** — 定义"哪些商品有资格被推荐"。运营手动维护的商品 ID 清单，系统内置全量池（ALL），商品货架实时同步全部可售商品。
- **第二层：排序策略** — 定义"候选商品按什么规则排序"。引用一个选品池作为候选来源，支持热销/新品/人工定序三种排序，可设置兜底策略。
- **第三层：策略组合** — 编排"页面上有几个资源位、每个资源位用什么策略"。包含 N 个资源位（最多 10 个），每个资源位绑定一个排序策略。
- **第四层：投放计划** — 控制"何时上线、投给谁"。绑定一个策略组合，设置生效时间、门店范围、人群范围，可开启 ABTest（用户级分流）。

一句话：**选品池**决定"推什么" → **排序策略**决定"怎么排" → **策略组合**决定"摆在哪" → **投放计划**决定"何时给谁看"

### 1.3 术语定义
- **选品池（召回池）**：可推荐商品的候选集合
- **SPU**：标准产品单元，商品最小管理粒度
- **全量池（ALL）**：系统内置选品池，商品货架实时同步全部可售商品
- **排序策略**：对选品池商品排序的计算单元
- **策略组合**：N 个资源位的编排方案，每个资源位绑定一个排序策略
- **资源位**：推荐页面上的一个商品展示位
- **投放计划**：推荐上线的唯一入口，控制时间/门店/人群/ABTest
- **Session 重排**：同一次请求内的去重、兜底、品类多样性控制

## 2. 后台导航结构
```
推荐中台
├── 选品管理
│   └── 商品选品池
├── 排序策略
│   └── 策略列表 / 策略编辑
├── 策略组合
├── 投放计划
├── 效果监控
└── 工具 & 诊断
    └── 全链路预览
```

## 3. 第一层：选品池管理

### 3.1 选品池列表页
**位置**：左侧导航「选品管理」→「商品选品池」
**布局**：Master-Detail（左列表 + 右详情预览）

**左侧列表区**：
- 顶部搜索栏：输入关键词实时搜索选品池名称，无结果时显示"未找到匹配的选品池"
- 搜索栏右侧：「新建选品池」按钮（主按钮）
- 列表项信息：
  - 选品池名称（最多 20 字，超出截断加"..."）
  - 状态标签：启用（绿色）/ 停用（灰色）
  - 商品数量：如"128 个商品"
  - 创建时间：YYYY-MM-DD
  - 引用数：被多少个排序策略引用
- 全量池（ALL）始终置顶，带"系统"蓝色角标，不可删除/停用/编辑
- 列表为空时：空态插画 + "还没有选品池，点击上方按钮创建第一个"

**右侧详情预览区**：
- 未选中时：引导态"选择左侧选品池查看详情"
- 选中后展示该池的详情摘要

**排序**：默认按更新时间倒序，全量池置顶
**数据加载**：骨架屏 → 无限滚动（每次 20 条）→ "已经到底啦" → 断网"网络异常，点击重试"

**交互逻辑**：
- 搜索：placeholder "搜索选品池"，输入后 300ms 防抖触发，超时 3s 提示"搜索失败，请重试"
- 列表项操作菜单（⋯按钮）：启用/停用、编辑、删除。全量池仅"查看详情"
- 停用确认：弹窗"停用后，引用该选品池的排序策略将无法获取候选商品。确定停用？"
- 删除：有引用→"该选品池被 {N} 个排序策略引用，无法删除"；无引用→二次确认→删除

### 3.2 选品池详情/编辑页
**顶部信息区**：选品池名称（点击可 inline 编辑，1-30 字）、状态标签、创建人/创建时间/更新时间、描述（最多 200 字）

**操作栏**：「添加商品」（主）+「批量导入」（次）+ 启用/停用开关。全量池隐藏所有编辑按钮。

**商品列表表格**：
| 列 | 说明 |
|---|---|
| 商品图片 | 40x40 缩略图，无图灰色占位 |
| SPU ID | — |
| 商品名称 | 最多 30 字截断 |
| 品类 | — |
| 状态 | 在售/下架/停售（只读，来源商品中心） |
| 添加时间 | — |
| 操作 | 移除 |

表头支持排序。表格上方：商品总数 + 搜索框。分页每页 50 条。

**引用关系区**（折叠面板）：展示引用该池的排序策略列表，可点击跳转。

**添加商品（单个）**：点击→侧滑面板→输入 SPU ID 或搜索→勾选→确认添加→loading→成功轻提示

**批量导入**：弹窗→文本框粘贴 SPU ID（每行一个或逗号分隔）→去重+无效提示+已存在跳过→上限 500 个

**移除商品**：单个/批量勾选→确认→轻提示。仅解除关联。

### 3.3 新建选品池
侧滑面板，表单：
| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| 选品池名称 | 文本 | 是 | 1-30字，不允许纯空格，不允许重名 |
| 描述 | 多行文本 | 否 | 最多 200 字 |
| 初始状态 | 开关 | — | 默认启用 |

底部：「创建」+「创建并添加商品」+「取消」

## 4. 第二层：排序策略

### 4.1 排序策略列表页
**布局**：标准表格页
**筛选区**：排序方式（全部/热销/新品/人工定序）、状态（全部/启用/停用）、搜索框

**表格列**：
| 列 | 说明 |
|---|---|
| 策略名称 | 最多 30 字截断 |
| 排序方式 | 热销排行（橙HOT标签）/ 新品排行（蓝NEW标签）/ 人工定序（紫Manual标签） |
| 引用选品池 | 选品池名称，点击可跳转 |
| 兜底策略 | 兜底策略名称，无则"-" |
| 被引用数 | 被多少个策略组合资源位使用 |
| 状态 | 启用（绿）/ 停用（灰） |
| 更新时间 | YYYY-MM-DD HH:mm |
| 操作 | 编辑 / 复制 / 停用(启用) / 删除 |

### 4.2 排序策略编辑页
**表单**：
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 策略名称 | 文本 | 是 | 1-30 字，不允许重名 |
| 描述 | 多行文本 | 否 | 最多 200 字 |
| 引用选品池 | 下拉选择 | 是 | 从启用中的选品池选择，展示池名称+商品数 |
| 排序方式 | 单选 | 是 | 热销排行 / 新品排行 / 人工定序 |
| 兜底策略 | 下拉选择 | 否 | 从其他排序策略中选择，不能选自己 |
| 状态 | 开关 | — | 默认启用 |

**条件区域（根据排序方式动态展示）**：
- 热销排行：排序维度（销量/交易额）、时间窗口（近7/14/30天）、最大返回数（默认50，范围1-200）
- 新品排行：排序维度（上新时间，固定）、时间窗口、最大返回数
- 人工定序：可拖拽商品列表（序号+图片+名称+SPU ID+移除按钮），至少1个商品

**兜底策略防环检测**：实时检查循环引用（A→B→A），检测到环则下拉选项置灰

## 5. 第三层：策略组合

### 5.1 策略组合列表页
**表格列**：组合名称、资源位数量、被引用数、状态、更新时间、操作（编辑/复制/停用/删除）

### 5.2 策略组合编辑页
**基本信息**：组合名称（1-30字）、描述（200字）、状态开关

**资源位编排区**（核心）：
- 纵向卡片列表，支持拖拽排序
- 每个资源位卡片：序号（自动编号）+ 绑定排序策略下拉 + 前3个商品预览缩略图 + 删除按钮
- 「添加资源位」按钮（最多10个）
- 最少保留1个资源位

**去重规则配置区**（折叠面板）：
- Session 内去重开关（默认开启）
- 同品类连续展示上限：数字输入（默认不限制，可设1-5）

## 6. 第四层：投放计划

### 6.1 投放计划列表页
**筛选区**：状态（全部/草稿/已发布/已暂停/已结束）、搜索框

**表格列**：计划名称、绑定策略组合、状态、优先级、生效时间、门店范围、ABTest、更新时间、操作

**生命周期**：草稿→已发布→已暂停↔已发布→已结束（不可逆）

### 6.2 投放计划编辑页
**基础信息**：计划名称(1-50字)、描述(500字)、优先级(1-100)、投放场景(Q1仅点单页)

**策略组合绑定区**：下拉选择+预览（资源位卡片横向排列）

**生效时间区**：开始时间+结束时间（日期时间选择器）+长期有效复选框

**门店范围区**：全部门店(默认)/指定门店（区域筛选+搜索+批量导入）

**人群范围区**：全部用户(默认)/指定人群（CDP人群包下拉搜索）

**ABTest 配置区**（折叠面板）：
- 启用开关（默认关闭）
- 实验 Key 文本输入
- 分组配置表格：分组名称、VID、绑定策略组合、流量比例
- 流量总和必须=100%
- 最少2组，最多5组

**底部操作**：「保存为草稿」+「保存并发布」（触发冲突检测）+「取消」

**冲突检测**：发布时检查同优先级+时间重叠+门店重叠的已发布计划，告警但不阻塞

## 7. 效果监控看板
**筛选区**：时间范围（今天/近7天/近30天/自定义）+ 投放计划多选

**核心指标卡片（3个）**：推荐位CTR、资源位空置率、推荐转化率（每个带环比变化↑绿/↓红）

**ABTest 对比区**（仅开启ABTest时）：实验组vs对照组并排展示+统计显著性

**趋势图**：折线图，CTR/空置率/转化率随时间变化

**选品池健康度表格**：名称、商品数量、有效商品比、被引用次数、健康状态（正常绿/注意黄/告警红）

## 8. 全链路预览
**输入区**：门店（搜索选择，必填）+ 用户ID（必填）+ 时间（默认当前，可修改）

**结果展示**：
- 命中信息：投放计划名称+优先级+ABTest分组+命中原因
- 资源位填充结果：横向卡片排列（序号+商品图片+名称+SPU ID），空资源位灰色虚线框+"未填充"
- Trace 信息（双视角Tab）：
  - 运营视角："资源位1展示了{商品名}，因为它在{选品池名}中，按热销排行排第1"
  - 技术视角：策略ID、排序算法、候选集大小、过滤原因、排名分数、去重跳过数

## 9. 全局规则
- 权限：v1.0不做权限管控，全量池系统级不可编辑
- 操作日志：v1.0不做审计日志，仅展示创建人和更新时间
- 超时：读5s，写10s，统一轻提示+按钮恢复
- 并发：乐观锁，冲突时提示刷新
- 自动过滤：排序策略运行时自动过滤下架/不可售商品
