# 推荐系统运营管理后台 — 前端开发 Spec

## 技术栈

- React + Ant Design / Arco Design
- React Router
- 图表：ECharts / Recharts
- 拖拽：dnd-kit / react-beautiful-dnd

## 全局布局

- 左侧固定导航栏（深色 #1a1a2e 或墨绿 #3a4a2e），宽 220px
- 右侧内容区，浅灰背景 #f5f6fa
- 顶栏：面包屑 + 用户头像

## 导航菜单结构

```
推荐中台
├── 选品管理 → /pools
├── 排序策略 → /strategies
├── 策略组合 → /combinations
├── 投放计划 → /plans
├── 效果监控 → /dashboard
└── 工具 & 诊断
    └── 全链路预览 → /preview
```

## 四层架构概念

```
第四层：投放计划    → 控制"何时上线、投给谁"
        ↑ 绑定
第三层：策略组合    → 编排"页面上有几个坑位、每个坑位用什么策略"
        ↑ 每个坑位绑定
第二层：排序策略    → 定义"候选商品按什么规则排序"
        ↑ 引用
第一层：选品池      → 定义"哪些商品有资格被推荐"
```

一句话：**选品池**决定"推什么" → **排序策略**决定"怎么排" → **策略组合**决定"摆在哪" → **投放计划**决定"何时给谁看"

## Page 1: 选品池列表 `/pools`

- 顶部：标题"商品选品池" + 搜索框 + "新建选品池"按钮
- 主体：卡片网格（每行 3-4 个），每张卡片显示：
  - 池名称、商品数量、创建时间、状态（启用/停用）
  - 操作：编辑、停用/启用、删除（有引用时禁用删除，tooltip 提示"该选品池正被 N 个排序策略引用，无法删除"）
- ALL 全量池置顶，标记"系统内置"徽章，不可编辑删除
- 空状态：插画 + "创建你的第一个选品池"引导按钮

## Page 2: 选品池详情 `/pools/:id`

- 顶部：池名称（可编辑）+ 状态开关 + 商品数量统计
- 操作栏：搜索商品 + "添加商品"按钮 + "批量导入"按钮
- **批量导入交互：**
  - 弹窗 Modal，textarea 输入区
  - 支持粘贴 SPU ID（逗号、换行、空格分隔均可）
  - 解析后展示预览列表（有效/无效/重复分组展示）
  - 确认后批量添加
- 商品表格列：缩略图、SPU ID、商品名称、品类、状态、添加时间、操作（移除）
- 支持 checkbox 多选 + 批量移除
- 底部分页
- ALL 全量池详情页：只读模式，不可添加/移除，说明文字"数仓 T+1 自动同步"

## Page 3: 排序策略列表 `/strategies`

- 顶部：标题"排序策略" + "新建策略"按钮
- 表格列：
  - 策略名称
  - 排序方式（彩色标签：热销=橙色、新品=绿色、人工=蓝色）
  - 引用选品池（名称，可点击跳转）
  - 被引用次数（数字，hover 展示引用方列表）
  - 兜底策略（名称或"无"）
  - 创建时间
  - 操作：编辑、复制、删除

## Page 4: 排序策略编辑 `/strategies/:id/edit`

- 表单字段：
  - **策略名称**（必填，文本输入）
  - **选择选品池**（下拉 + 搜索，展示池名称 + 商品数量）
  - **排序方式**（Radio Group）：
    - 热销排行（HOT）— 按销量/交易额自动排序
    - 新品排行（NEW）— 按上新时间排序
    - 人工定序（manual）— 切换后展示拖拽排序区域
  - **人工定序模式：**
    - 左侧：选品池商品列表（可搜索、多选添加）
    - 右侧：已选商品排序列表（拖拽排序，可移除）
  - **兜底策略**（下拉选择其他排序策略，可选，可清空）
    - 循环引用检测：选择后立即校验，检测到环时输入框变红 + 错误提示"检测到循环引用：A → B → A"
  - **过滤规则**：自动过滤下架/不可售商品（默认勾选，灰色不可取消，说明文字"系统自动过滤"）
- 底部：保存 + 取消

## Page 5: 策略组合列表 `/combinations`

- 顶部：标题"策略组合" + "新建组合"按钮
- 表格或卡片展示：
  - 组合名称
  - 坑位数
  - 关联投放计划数（可点击查看）
  - 状态（启用/停用）
  - 创建时间
  - 操作：编辑、复制、删除

## Page 6: 策略组合编辑 `/combinations/:id/edit`

> 对标选中的右下版设计稿

### 顶部统计卡片行（4 个指标卡片）
- 总坑位数
- 已绑策略数
- 关联投放计划数
- 预计覆盖商品数

### 左侧主区域 — 坑位配置表格
- 列：
  - 拖拽手柄（⠿ 图标）
  - 坑位序号（#1, #2, #3...自动编号）
  - 绑定策略（下拉选择排序策略，展示名称 + 排序方式标签）
  - 候选池来源（自动回显，灰色只读）
  - 预览商品（缩略图 + 商品名，实时从策略结果取第 N 个）
  - 操作（删除坑位）
- 底部："+ 添加坑位"按钮（最多 10 个，满后禁用）
- 拖拽调整顺序

### 右侧区域 — 关联信息
- 关联的投放计划卡片列表（名称 + 状态标签 + 生效时间）
- 无关联时显示空状态"暂无投放计划使用此组合"

### 底部 — 预览区
- 标题"推荐效果预览"
- 模拟点单页推荐 Tab：商品卡片横排滚动，每张卡片显示商品图 + 名称 + 价格
- 配置项：
  - 同品类最多连续展示数（数字输入，默认值"不限制"）
  - Session 内去重（开关，默认开启，灰色不可关闭，说明文字"同一次请求内不同坑位不展示重复商品"）

### 底部操作栏
- 保存 + 取消

## Page 7: 投放计划列表 `/plans`

- 顶部：标题"投放计划" + Tab 筛选（全部 / 草稿 / 已发布 / 已暂停 / 已结束）+ "新建计划"按钮
- 表格列：
  - 计划名称
  - 状态（彩色标签：草稿=灰色、已发布=绿色、已暂停=橙色、已结束=灰色描边）
  - 优先级（数字）
  - 生效时间（起止时间）
  - 门店范围（全部 / N 个区域 / N 家门店）
  - 绑定策略组合（名称，可点击跳转）
  - ABTest（开启/未开启 标签）
  - 操作：编辑、发布/暂停、复制、删除
- **冲突行**：黄色底色 + ⚠️ 告警图标，hover 提示冲突详情

## Page 8: 投放计划编辑 `/plans/:id/edit`

### 表单分区（Collapse / Anchor 锚点）

**① 基础信息**
- 计划名称（必填）
- 投放场景（下拉，Q1 仅"点单页推荐位"一个选项）

**② 策略配置**
- 绑定策略组合（下拉搜索选择，展示组合名称 + 坑位数）
- 版本号（自动生成，只读）

**③ 投放范围**
- 优先级（数字输入 1-99，说明文字"数字越大优先级越高，同一资源位多个计划时取第一个命中的"）
- 生效时间（日期时间范围选择器，开始 ~ 结束）
- 门店范围（Radio：全部门店 / 指定区域 / 指定门店）
  - 指定区域：多选下拉，搜索
  - 指定门店：多选下拉，搜索，支持粘贴门店 ID
- 人群范围（Radio：全部用户 / 指定人群包）
  - 指定人群包：多选下拉

**④ ABTest 配置**（可折叠区域，默认收起）
- 启用开关
- 启用后展示：
  - 实验 Key（文本输入，说明文字"火山引擎实验标识"）
  - 分组配置表格：
    - 列：分组名称、VID、流量比例（%）
    - "+ 添加分组"按钮
  - 底部说明："分流由火山引擎执行，系统仅存储配置信息并在运行时透传。分流粒度为用户级（user_id），同一门店内不同用户可能看到不同内容。"

**⑤ 冲突检测**
- 保存时自动检测：同优先级 + 同门店 + 同时间段
- 有冲突时弹窗告警，展示冲突计划列表，按钮："仍然保存" / "返回修改"

### 底部操作栏
- 生命周期按钮组：保存草稿 / 发布 / 暂停（根据当前状态动态显示）

## Page 9: 效果监控 `/dashboard`

- **顶部筛选栏：**
  - 时间范围（快捷选择：今天/近7天/近30天 + 自定义）
  - 投放计划选择（下拉多选）
  - 门店筛选（下拉多选）

- **指标卡片行（3 个）：**
  - 推荐位 CTR — 数字 + 趋势箭头（↑绿/↓红）+ 环比
  - 坑位空置率 — 数字 + 趋势箭头 + 环比
  - 曝光量 — 数字 + 趋势

- **趋势图区域：**
  - 折线图，X 轴时间，Y 轴指标值
  - 可切换查看不同指标
  - 支持多计划叠加对比

- **ABTest 对比区（启用时显示）：**
  - 标题"实验效果对比"
  - 表格：分组名称 | CTR | 空置率 | 曝光量 | 显著性标记
  - 支持"查看详情"展开每日数据

## Page 10: 全链路预览 `/preview`

- **输入区：**
  - 门店选择（下拉搜索）
  - 用户 ID（文本输入）
  - "模拟请求"按钮（主按钮样式）

- **结果区上半部 — 推荐结果：**
  - 命中信息条：投放计划名称 + ABTest 分组（如有）
  - 坑位结果：横排商品卡片，每张显示：
    - 商品缩略图
    - 商品名称
    - 价格
    - 坑位编号角标（#1, #2, #3...）
  - 未命中任何计划时显示空状态 + 原因说明

- **结果区下半部 — Trace 信息（Tab 切换）：**
  - **运营视角 Tab：**
    - 逐坑位说明，自然语言："坑位 1 → 推荐了「多肉葡萄」→ 原因：热销排行第 1 名"
    - 标记跳过的商品及原因（如"去重跳过"、"已下架跳过"）
  - **技术视角 Tab：**
    - JSON 风格结构化展示
    - 字段：slot_id、strategy_id、pool_id、rank、skip_count、fallback_used、latency_ms

## Mock 数据说明

商品使用茶百道产品线：
- 奶茶类：豆乳玉麒麟、茉莉奶绿、芋圆奶茶、黑糖珍珠奶茶
- 果茶类：多肉葡萄、超级杯水果茶、满杯橙果、西瓜啵啵
- 咖啡类：生椰拿铁、美式咖啡、燕麦拿铁
- 新品：当季限定款

门店使用成都区域：
- 春熙路太古里店、天府广场店、高新区银泰店、双流机场店等

人群包：
- 高频用户（月消费 ≥ 8 次）
- 新注册用户（30天内）
- 沉睡用户（60天未消费）
- 咖啡偏好用户

## 设计风格

延续 Agent 引擎设计方案风格（可调整）：
- 导航栏：墨绿 #3a4a2e 或深蓝 #1a1a2e
- 强调色：蓝色 #1664ff
- 背景色：浅灰 #f5f6fa
- 卡片：白色 #ffffff，圆角 8px，轻投影
- 字体：系统默认，标题 18px，正文 14px，辅助 12px
